### DevOpsエンジニア  
**2023年10月 – 2024年3月**  
**アクセンチュア（東京、日本）**  

### 1. 案件概要
金融業界のクライアントにおける大規模システム用CI/CDパイプライン構築に参画。  
チーム規模：7名（PM1名、エンジニア6名）

### 2. 担当業務
- デプロイステージに必要な定義ファイルの作成
- Pythonスクリプトによる資材調査・データマッピングの自動化
- Excel VBAを活用したマッピングドキュメントの作成
- Angularアプリケーションのビルド・デプロイステージの実装
- 結合テスト・システムテストの実施およびPytestによる自動テスト導入
- GitLabへのデプロイ先定義ファイルの一括登録作業の自動化

### 3. 実績
#### (1) 資材調査・マッピングファイル作成
- **Jarファイルのデプロイ先調査:**  
要件定義フェーズでは、1万件を超えるJarファイルについて、本番環境での正しいデプロイ先を特定し、CI/CDによる自動デプロイに対応する定義ファイルを作成。約1万のJarファイルを約80のデプロイ先ディレクトリに正確に振り分けるため、Pythonを用いて情報を整理・マッピングし、最終的にJSON形式のファイルとして出力した。

- **Excel VBAによる可視化**  
Pythonスクリプトの出力JSONを基に、リポジトリ⇔本番環境のJar配置状況を一覧化できるようなExcelマクロを作成。差分のあるJarの調査と報告資料作成の効率化に貢献。

#### (2) CI/CDパイプライン実装:
- **AngularアプリのCI/CDステージ設計**  
Angularアプリケーションについて、ビルド・デプロイステージを構築。共通処理をPythonスクリプトに実装し、ymlから呼び出す形式を採用したことで、再利用性の向上に貢献。

#### (3) テスト対応・品質担保
- **結合テスト・システムテストの実施**  
リリースに向けたテストフェーズではテストケースの作成と打鍵を担当し、バグの検知に貢献。加えて、Pythonベースの自動テストツールPytestを導入し、バグの早期発見を実現。

#### (4) 本番リリース対応
- **Jar配置定義ファイルのGitLab登録自動化**  
本番リリースに向け、約80のリポジトリに対して定義ファイルを登録する必要があったため、Excelで一覧を作成し、Git操作（pull・commit・push）コマンドを関数で生成する仕組みを構築。Git Bashに貼り付けるだけで作業できる仕組みを整備し、工数とヒューマンエラーの削減を実現。

### 4. 使用技術
GitLab | GitLab CI/CD | Python | Shell | Docker | JIRA
